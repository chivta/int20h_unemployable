

# Сценарій

Після вірусного зростання й розв'язання проблем із податковою компанія зіткнулася з новими викликами масштабування. Виявилося, що попри сильний "перший контакт" компанія майже нічого не знає про мотивацію та реальні потреби користувачів.

Користувачі приходять у продукт через різні маркетингові канали — зокрема рекламу в Instagram та Facebook. Найчастіше це відбувається з телефону, а найшвидша й найорганічніша точка контакту — in-app browser. Веб-платформа дозволяє почати взаємодію миттєво, не змушуючи користувача встановлювати застосунок.

# 2\. Функціональні вимоги

## *User Flow*

Користувач клікає на рекламу

Користувацька частина (User-facing) веб\-платформи відкривається в in-app browser

Користувач відповідає на низку питань, де кожен наступний крок залежить від попередніх відповідей

На основі зібраної інформації платформа формує персоналізований офер

# **3.1. Загальна модель**

1. Потік питань та інформаційних сторінок (info pages) утворює **directed acyclic graph (DAG)**: вузли та зв'язки, без циклів. Типи вузлів:

   1. **Вузол питання (Question node):** single choice / multi choice / input (число/текст)

      2. **Інформаційний вузол (Info page node):** інформаційна/мотиваційна сторінка ("message"), без збору даних

      2. Кожен користувач унікальний, тож щоб не задавати всім однакові питання, система має вміти будувати розгалуження на основі зібраних даних.

      3. Кількість вузлів і розгалужень потенційно необмежена

      4. Розгалуження можуть базуватись на декількох атрибутах одночасно (наприклад, чоловік 25-40 років, захоплюється бігом, має травму коліна)

      5. Варіанти відповідей на однакові питання можуть відрізнятись залежно від попередніх відповідей користувача.

      6. На основі зібраних даних користувачу видається персоналізований офер, як от: план тренувань по визначеному найбільш релевантному топіку (заняття в залі з коачем, план тренувань для бігу або підготовки для марафону, йога вдома, функціональні вправи з власною вагою, план для схуднення

## 3.2.1. Адмінка

### **3.2.1.1** Admin FE — редактор графа

* Візуальний редактор, який дозволяє:  
  * створювати/редагувати Question nodes та Info page nodes  
  * створювати/видаляти edges між nodes  
  * налаштовувати умови переходів (rules/conditions), щоб було видно “як вузли ведуть до наступних”

### **3.2.1.2** Admin BE — керування конфігурацією

* Зберігає і версіонує (опційно) конфігурацію графа.  
* Віддає конфігурацію як raw content (конфіг): структура DAG, тексти, опції, правила — без рендерингу сторінок.

### **3.2.1.3** Content Delivery API

* Endpoints, які дозволяють User-facing частині отримати актуальну конфігурацію (raw content) для рендерингу.

## 3.2.2. User App

### **3.2.2.1** User FE — рендеринг і проходження flow

* Відкривається в in-app browser.  
* Отримує raw content (DAG) через Content Delivery API.  
* Рендерить Question nodes та Info page nodes.  
* Збирає відповіді користувача та веде його по графу (відповідно до правил переходів).

### **3.2.2.2** User BE — збереження відповідей і рішення про офер 

* Зберігає:  
  * сесію/контекст користувача  
  * відповіді на питання  
  * похідні атрибути від відповідей за потреби  
* На основі зібраного контексту приймає рішення про **основний офер** **(primary offer)**:  
  * **digital subscription plan \+ physical wellness kit**  
* Видає фінальний результат для відображення у User FE (offer \+ коротке “чому” \+ CTA).


# **3.3. Вхідні і вихідні дані**

Якщо подивитися ширше, система має дозволяти налаштовувати “шляхи” користувачів — такі послідовності питань, які, на нашу думку, найкраще розкривають потреби користувача та підвищують імовірність того, що він оплатить фінальний офер.

Важливо: різні послідовності відповідей можуть приводити до одного й того самого офера або до двох оферів одразу. Наприклад:

* Послідовність відповідей користувача 1 → **Офер 1**  
* Послідовність відповідей користувача 2 → **Офер 1**  
* Послідовність відповідей користувача 3 → **Офер 2**  
* Послідовність відповідей користувача 4 → **Офер 1, Офер 2**

Ми пропонуємо орієнтуватися на запропоновані в задачі атрибути користувача та каталог оферів, із можливістю додати власні варіанти. Використовувати всі запропоновані атрибути й офери не обов’язково — головне, щоб рішення було логічним, послідовним і креативним.

## **3.3.1. Атрибути користувача для питань**

Набір сигналів/атрибутів, які необхідно **уточнити, розвинути та формалізувати** (зокрема визначити можливі значення, формат і спосіб збору через питання):

* **Age:** вік (або віковий діапазон)  
* **Gender:** стать (за потреби — опція “не вказувати”)  
* **Goal:** схуднення / сила / витривалість / гнучкість / зниження стресу  
* **Context:** дім / зал / outdoor; наявність обладнання  
* **Constraints:** час, графік, травми/обмеження  
* **Level:** beginner / intermediate / advanced  
* **Motivation & barriers:** “чому зараз”, що заважає (стрес, втома, брак часу/дисципліни)  
* **Preferences:** інтенсивність, формат (короткі сесії, йога, біг, силові)  
* **Self-reported wellbeing (самооцінка стану):** стрес / сон / енергія

## **3.3.2. Каталог оферів**

1. **Weight Loss Starter (Home) — 4 тижні**  
    Digital: план схуднення вдома (20–30 хв)  
    Physical wellness kit: *Home Fat-Burn Kit* (resistance bands, скакалка, шейкер/пляшка, електроліти \+ healthy snack)

2. **Lean Strength Builder (Gym) — силові \+ прогресія**  
    Digital: програма для залу  
    Physical wellness kit: *Gym Support Kit* (wrist wraps/straps, mini loop band, компактний рушник, електроліти/протеїн-снек)

3. **Low-Impact Fat Burn — “суглоби friendly”**  
    Digital: low-impact план (коліна/спина friendly)  
    Physical wellness kit: *Joint-Friendly Kit* (knee sleeve/бандаж, massage ball, mini loop bands, cooling patch/recovery gel)

4. **Run Your First 5K (Outdoor) — бігова програма**  
    Digital: підготовка до 5K (3 рази/тиж)  
    Physical wellness kit: *Runner Starter Kit* (electrolytes, reflective armband/safety light, blister kit, running belt)

5. **Yoga & Mobility (Home) — гнучкість \+ спина/постава**  
    Digital: йога/мобільність 10–25 хв  
    Physical wellness kit: *Mobility Kit* (travel yoga mat або yoga strap, massage ball, mini foam roller)

6. **Stress Reset Program — ментальний ресет \+ мікрозвички**  
    Digital: дихання/медитації/антистрес рутини  
    Physical wellness kit: *Calm-Now Kit* (eye mask, aroma roll-on/mini candle, tea sticks, stress ball/fidget, quick reset card)

7. **Quick Fit Micro-Workouts — 10–15 хв щодня**  
    Digital: короткі щоденні тренування  
    Physical wellness kit: *Micro-Workout Kit* (slider discs, mini loop bands, шейкер/пляшка, mini routine card)
